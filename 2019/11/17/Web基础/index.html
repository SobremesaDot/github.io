Internet & Intranet 网络

  • Internet
    • 定义
      • 国际互联网
      • 由那些使用公用语言互相通信的计算机连接而成的全球网络
    • 起源：
      • 美国国防部高级研究计划局的前身ARPA建立的ARPAnet，该网于1969年投入使用。
    • 目的：允许世界上数以亿计的人们进行通讯和共享信息
  • Intranet
    • 定义：企业内部网
    • 特点：
      • 相对封闭的网络环境
      • 防火墙保护
      • 有权限控制
    • 目的：使企业内部的秘密或敏感信息收到保护

三种网络体系架构

  • C/S架构

    • 全称：Client/Server，即客户端/服务器
    • 又称为胖客户端。例如：360安全卫士、手机APP
    • 优点：
      • 客户端与服务器直接相连，响应速度快
      • 界面多样性、满足客户个性化要求
      • 能实现复杂的业务流程
      • 面对有限群体，具有高机密性
        • 缺点：
        • 客户端安装程序，不能够实现快速部署安装和配置
        • 兼容性差，各操作系统、多版本都要面对
        • 升级成本较高

  • B/S架构

    • 全称：Browser/Server，即浏览器/客户端
    • 又称为瘦客户端
      • 客户端使用浏览器
      • 服务器端实现事务逻辑
      • 实例：购物网站网页版
    • 优点：
      • 具有分布性特点，可以随时随地进行查询、浏览等业务处理
      • 业务扩展简单方便，通过增加网页即可增加服务器功能
      • 维护简单方便，只需要改变网页，即可实现所有用户的同步更新
    • 缺点：
      • 个性化特点明显降低，无法实现具有个性化功能的要求
      • 操作是以鼠标为最基本的操作方式，无法满足快速操作的要求
      • 页面动态刷新，响应速度明显降低

  • P2P架构

    • 全称：Point-to-Point
    • 点对点。实例：飞秋、QQ直播、迅雷
    • 优点：速度快
    • 缺点占用系统资源多

Web的特点

  • 图形化、易于导航
  • 与平台无关
  • 分布式
  • 动态
  • 具有交互性

Web工作原理

  • HTTP处理流程
    1. 建立连接
    2. 客户端浏览器发送请求给Web服务器
    3. Web服务器解析请求找到相应的资源
    4. Web服务器生成响应返回客户端
    5. 关闭连接
  • 客户端和服务器工作原理

Web站点的组成部分

  • 实例：
    • http://quotes.money.163.com/fund/jzzs_169101.html?start=2019-01-02&end=2019-01-31
    • 协议类型://主机名:端口号/路径/页面文件?附加部分
  • 协议类型
    • http/https
    • ftp
  • 主机名
    • 域名和IP等价。实例：https://www.baidu.com/ = https://180.97.33.107/
    • DNS服务器把域名解析为IP
  • 端口号
    • 默认端口号可以省略。如：如https://www.baidu.com = https://www.baidu.com:443
      • http默认端口80
      • https默认端口43
      • ftp默认端口21
    • 非默认端口要注明端口号
  • 页面文件
    • 服务器上哪个文件，包括路径
    • 默认文件可以忽略。一般为index.htm或者是index.php
  • 附件部分
    • URL = Uniform Resource Locator。统一资源定位符
    • URL地址参数
      • ？之后的部分。如：start=2019-01-02&end=2019-01-31
